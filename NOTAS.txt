
COSTOS MATERIA PRIMA
COSTOS OPERACIONES

SE NECESITA COMPARAR ENTRE MANUAL Y AUTOMATICA, QUE SON LAS CENTRALES DE MEZCLAS.

PERTENECEN A UNA CENTRAL DE MEZCLAS
- LINEAS DE PRODUCCION
- PRODUCCION DIA (O POR DIA)
- PRODUCCION MENSUAL

LOS PORCENTAJES PERTENECEN A CADA BLOQUE DE MATERIA PRIMA: ADULTO, NEONATAL Y PEDIATRICA.

PERO DEBEN ESTAR A NIVEL DE LA CENTRAL. QUEDA POR ANALIZAR Y DEFINIR.


ENTONCES UNA CALCULADORA, SERIA UNA POR CADA TIPO DE CENTRAL. *

CALCULADORA
- AUTOMATICA
    - ADULTO
    - NEONATAL
    - PEDIATRICA
- MANUAL (POR DEFECTO)
    - ADULTO (POR DEFECTO)
    - NEONATAL
    - PEDIATRICA
- CONFIGURACION
    - VARIOS MODELOS.
- TOTALES

TODO EL ENFOQUE ANTERIOR HACE QUE SE VUELVA MUY GRANDE EL MODELO.

LOCALFORAGE PERMITE MULTIPLES VALORES, YA QUE SE ALMACENA UN KEY Y UN JSON.

Y EL TRUCO ESTARIA EN LOS NOMBRES A UTILIZAR!!!

ENTONCES YA QUE EL MODELO CENTRAL DE MEZCLAS TIENE SUB MODELOS DEFINIDOS Y HAY LLAVES CLARAS
QUE PUEDEN GENERAR UNA LLAVE COMPUETA, LA IDEA SERIA QUE CADA BLOQUE O SUBCOMPONENTE DE
REACT SOLO DEPENDA DE LAS LLAVES PRIMARIAS GENERADAS A PARTIR DEL NOMBRE DEL COMPONENTE EN SI.

TENEMOS COMO LLAVES PRIMARIAS
- TIPO DE CENTRAL
- TIPO DE POBLACION

DE ESA MANERA SE GENERARIA ALGO ASI:

- DEFAULT:MANUAL:ADULTO:COMPONENTE_1
- CURRENT:AUTOMATICA:NEONATAL:COMPONENTE_2

ENTONCES EL COMPONENTE RECIBE LOS VALORES POR DEFECTO, MAS LAS LLAVES PRIMARIAS, Y MAS LA LLAVE DEL
COMPONENTE EN SI, SE GENERA LA LLAVE COMPUESTA Y CARGA LA DATA DESDE LA DB LOCAL.

ASI LA PROPAGACION HACIA EL PADRE DEPENDE DE UN SOLO EVENTO SIMPLE, Y EL CUAL SE DISPARA CON EL PROMISE DEL LOCALFORAGE.

QUIZAS LO MAS COMPLEJO SERIA LA LOGICA DE COMPARAR RESULTADOS, LO CUAL TENDRIA QUE CARGAR TODOS LOS DATOS DESDE LA DB LOCAL.

EL MODELO "CENTRAL DE MEZCLAS" QUE AGRUPA MULTIPLES ELEMENTOS, ES EL MODELO QUE SE DE USAR PARA SALVAR LOS CALCULOS CON UN
NOMBRE POR USUARIO.

ES EL MODELO CONTENEDOR DE LOS DATOS PARA REALIZAR LAS COMPARACIONES.

AHORA PARA TENER UNA LISTA DE LOS CALCULOS GUARDADOS EN LA DB DEBE TENER UNA ENTRADA LLAMADA "CALCULOS USUARIO", ENTONCES 
UN ELEMENTO DE ESA LISTA CORRESPONDE A LA LLAVE PRIMARIA O LLAVE INICIAL DE LA LLAVE COMPUESTA.

POR EJEMPLO:
MIARCHIVO:FILENAME:CURRENT:MANUAL:ADULTO:COMPONENTE_1 0 ALGO ASI... ESTA POR DEFINIR
