
YA QUE PARA EVITAR EL EXCESO DE MODELOS SE USAN LOS MISMO MODELOS PARA LAS DISTINTAS CENTRALES. QUE HACER ?

PUES HAY QUE ALMACENAR LOS DATOS EN LA DB LOCAL.

DE ESA MANERA LA SOBRECARGA DE MEMORIA ES MENOR Y LA LOGICA MENOS PESADA.

AQUI ES DONDE ENTRA EN JUEGO LOS ESQUEMAS DE LA DB:

DEFAULT:
- SIGNIFICA QUE LOS VALORES CON ESTA LLAVE, SON LOS VALORES DEFINIDOS POR DEFECTO, O QUE EL USUARIO LOS MODIFICO PARA QUE SE ACTUALIZARAN. AHORA BIEN, 
- DIGAMOS QUE ESTAN LO SIGUIENTE, DEFAULTS DE FABRICA, Y DEFAULTS USUARIO. SOLO PUEDE EXISTIR UNO, POR CADA CALCULADORA.
- AHORA BIEN, QUEDA LA DUDA SI LOS VALORES DEFAULT PUEDEN SER A NIVEL GLOBALES. <-- **** PREGUNTA COMPLEJA, PERO QUEDARIA PARA UNA SIGUIENTE VERSION.


CURRENT:

SIGNICIA QUE LOS VALORES TEMPORALES, DE LA CALCULADORA DRAFT, SE ALMACENAN ACA, Y SON BORRADOS CUANDO SE USAN TODAS LAS OPCIONES DEL MENU CALCULADORA.
ESTO PORQUE NO DEBEN QUEDAR CON ESTE NOMBRE.

IMPORTANTE, TOCA CALCULADORA TIENE *** AUTOSALVADO *** YA SEA COMO CURRENT O CON EL NOMBRE DE ARCHIVO QUE LE DIERON.


* ENTONCES CUALQUIER CALCULADORA SIN SALVAR, OSEA QUE SOLO SE ESTA MANEJANDO EN MEMORIA, SE ESTARA ALMACENANDO TEMPORALMENTE EN LA DB LOCAL.
CON LA FINALIDAD DE HACER UN SWAPPING DE LOS DATOS DE LAS DISTINTAS CENTRALES.
ESTO DEBE SER A HUEVO, PARA TENER PERSISTENCIA DE DATOS DURANTE EL USO DE LA CALCULADORA.


FILENAME:

- UNA VEZ QUE EL USUARIO DEFINE UN NOMBRE DE ARCHIVO, TODOS LOS DATOS QUE ESTAN ALMACENADOS CON LA LLAVE * CURRENT * SE VUELCAN CON EL NOMBRE DEFINIDO POR EL USUARIO.
Y LO MAS IMPORTANTE QUE PERO SE SIGUE TRABAJANDO SOBRE * CURRENT *, PORQUE SINO CADA CAMBIO ESTARIA ACTUALIZANDO EL ARCHIVO, Y SI EL USUARIO SE EQUIVOCA Y NO QUIERE 
ESO CAMBIOS, YA ESTARIA ACTUALIZADO EL FILE Y NO SE QUIERE. EL USUARIO ESTA OBLIGADO A USAR LA OPCION "SALVA", TAL Y COMO QUE SE HACE EN MS-WORD (QUE ES UN STANDARD).

- TAMBIEN, SI EL USUARIO ABRE UN FILE, Y LUEGO QUIERE SALVARLO CON UN NUEVO NOMBRE, ESTARIAMOS HABLANDO QUE SERIA EL MISMO PROCESO DE VOLCADO, YA QUE LOS DATOS ESTARIA
DENTRO DE * CURRENT *.

* SI EL USUARIO UTILIZA LA FUNCIONA SALVAR O SALVAR COMO, EL PROCESO ESTARIA TOMANDO TODOS LOS DATOS ALMACENADOS TEMPORALMENTE Y VOLCANDOLOS 
EN REGISTROS NUEVOS.




SI EL USUARIO, YA NO QUIERE LOS DATOS ACTUALES EN MEMORIA Y NO DESEA GUARDAR LOS CAMBIOS, VA A USAR LA OPCION "NUEVA CALCULADORA" LO QUE SIGNIFICA
QUE EL SISTEMA ESTARIA BORRANDO TODOS LOS DATOS ALMACENADOS TEMPORALMENTE.

PERO AHI EN DONDE COMIENZA LO DIVERTIDO, PORQUE HAY CONDICIONES QUE RESPETAR.
